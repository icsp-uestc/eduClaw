## EduClaw

EduPilot本质上是基于智能体的Web应用：利用大语言模型、检索增强生成和功能调用等技术，将传统的**分数评价**转变为量化的**能力画像评价**，为学生提供自然语言交互式的选课路径模拟与规划定制服务。

而OpenClaw则是一个具备**全天候后台常驻运行**、**支持跨通讯软件原生接入**以及**拥有持久化本地记忆体系**的底层智能体运行与编排框架。

使用OpenClaw作为底层架构，可以将单节点的EduPilot升级为具备长期记忆连贯性、能够脱离网页限制融入日常通讯软件，并可以通过配置定时任务机制主动发起学业预警与干预的全自动、泛在伴学式基础设施。

我们暂时将其命名为**EduClaw**，以下是**EduClaw**开发流程提纲。

### 一、 架构改造与底层迁移

1. **大模型底座与高性能本地部署**
   
* 为了保障庞大学生数据的隐私并降低长期运行的成本，放弃纯云端 API，转而使用 `vLLM` 部署本地的开源大语言模型（如 Qwen 系列）。将本地 `vLLM` 服务直接通过 OpenClaw 的 Gateway 接入，作为 Agent 的推理大脑，支撑高并发的复杂学业请求。

2. **交互入口重构**
   
* 将原有的Web端降级为单纯的复杂图表渲染引擎（如展示多维能力雷达图）。日常的“互动探索”和“路径模拟” 全部迁移至 OpenClaw 原生支持的通讯平台（如 Telegram 或飞书），实现真正的自然语言无缝对话。

3. **持久化记忆注入**
   
* 将学生每学期上传的成绩单、历史规划方案 转化为 Markdown 格式，存入 OpenClaw 的本地记忆库。这样跨学期的对话也能保持绝对连贯，实现“规划-执行-评估-再规划”的无缝闭环。


### 二、 核心教学功能技能化

OpenClaw 依靠 Skills（工具/插件）来与外部世界交互，为了使得EduClaw具备EduPilot的原生功能，需要将原有系统的核心模块封装为 Agentic Skills：

1. **能力画像生成 Skill**
* 编写 Python 脚本作为调用工具，解析学生上传的成绩数据，调用内部的深度学习映射模型，计算出指标得分并更新到学生的本地记忆档案中。


2. **多智能体选课检索 Skill**
* EduPilot使用了基础的 RAG 技术进行课程检索。在这里可以进一步升级，使用 LangGraph 构建一个专门的数据库查询子工作流。当学生提出复杂选课需求时，主 Agent 调用该 Skill，内部通过 NL2SQL 技术将自然语言精准转化为 SQL，从课程数据库中提取出最优的培养路径并返回给用户。



### 三、 衍生服务设计

依托 OpenClaw 的主动运行与自动化能力，可以衍生出传统教学系统无法做到的高价值服务：

1. **主动式学业预警与伴学**

* 利用 OpenClaw 的定时任务机制，让 Agent 每天在后台自动比对学生的学业进度。当发现某项能力指标（如“设计开发能力”）长期未达标，或者核心课程即将考核时，EduClaw 会主动发消息给学生推送补救方案。


2. **非结构化能力数据采集**

* EduPilot高度依赖成绩单，难以纳入非分数成果。EduClaw可以通过日常对话收集学生的隐性经历。例如，当学生在聊天中提到自己假期参与了某某社会实践活动时，Agent 能够在后台自动触发分析，提取出真实的“沟通协作”或“工程问题解决”能力 的证据链，并量化补充进雷达图中。


3. **基于偏好的对齐优化飞轮**
* 在学生与 EduClaw 的交互过程中，收集他们对于推演出的学习路径的采纳率或修改意见。将这些交互日志作为偏好数据，在后台定期使用 GRPO 或 DPO 算法对底层的教育大模型进行强化学习微调对齐，让 Agent 的推荐决策越来越契合真实的专业培养目标。


4. **安全与人机回环机制**
* 针对大模型的幻觉风险，可以开发一个“审批 Skill”：EduClaw 生成的最终个性化学期选课方案在发送前，先通过内部接口推送给真实的教学专家或导师。专家点击确认后，再由 Agent 正式发送给学生执行。
